# Regra Visual do RPG FUSHI

Estas regras sao contexto de direcao visual. Nao significam migracao imediata.

## CSS / Framer Motion

Usar apenas para interface:

- paineis
- botoes
- janelas
- cards
- modais
- logs
- transicoes de UI

Nao usar CSS seco para efeitos principais de magia, clima, dominio ou combate.

## SVG

Usar para elementos vetoriais limpos:

- icones
- selos
- rotas
- cones
- areas de habilidade
- simbolos de faccao
- circulos de dominio simples

## Canvas / PixiJS

Usar para o MAP 2D:

- tokens
- grid
- camadas
- particulas
- auras
- efeitos de FUSHI
- clima
- nevoa de guerra
- areas animadas
- impacto visual

## WebGL / Pixi Filters

Usar para efeitos avancados 2D:

- glow
- blur
- distorcao
- glitch
- sombras vivas
- energia organica
- dominio distorcendo mapa

## Three.js

Usar somente para cenas especiais:

- portais
- dominio cinematografico
- dragao FUSHI
- ilha 3D
- eventos cataclismicos
- cutscenes especiais

Nao usar Three.js para tudo.

## Howler.js

Usar para audio real com samples.

Nao gerar chuva, fogo ou ambiente com oscilador sintetico simples.

Usar loops reais:

- chuva
- vento
- caverna
- floresta
- fogo
- mar
- tensao
- dominio
- FUSHI escuro

## Assets

Usar assets para:

- mapas finais
- retratos
- tokens
- icones importantes
- sons reais
- texturas especiais

Usar programacao para:

- brilho
- particulas
- aura
- transicao
- distorcao
- area
- movimento
- feedback visual

## Objetivo

Nada de placeholder generico com bolinha aleatoria como resultado final.

Todo efeito deve ter intencao visual, parametros e relacao com o FUSHI.
